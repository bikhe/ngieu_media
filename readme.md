# Биржа СМИ

Комплексная система управления медиацентром (фотографы, видеографы, пилоты БПЛА). Позволяет организаторам создавать заявки на съемку, бронировать технику, а исполнителям — брать задачи в работу через нативное мобильное приложение.

## Технологический стек

Система состоит из трех независимых модулей, объединенных REST API:

* **Backend:** Python 3.11, Django, Django REST Framework, PostgreSQL.
* **Web Admin (Фронтенд):** React, TypeScript, Material UI (MUI v6).
* **Mobile App (Мобилка):** Flutter, Dart (Android / iOS).
* **Инфраструктура:** Docker, Docker Compose, Nginx.

## Основные возможности

* **Система ролей и допусков:** Администраторы, Организаторы и СМИ. Доступ к сложным съемкам регулируется скилл-уровнями (PRO, VIDEO, DRONE).
* **Бронирование техники:** Интегрированный склад оборудования. При взятии задачи исполнитель может "закрепить" за собой нужную камеру или петлички.
* **Чат мероприятий:** Встроенный мессенджер внутри каждой заявки для обсуждения ТЗ и локаций.
* **Telegram Интеграция:** Автоматические уведомления организаторам о том, что их задачу взяли в работу или прислали сообщение в чат.
* **Feature Toggles:** Возможность "на лету" отключать модули (чат, технику, жесткие дедлайны) через настройки бэкенда.
* **Экспорт отчетов:** Выгрузка всей статистики по мероприятиям в CSV.

## Быстрый старт (Развертывание через Docker)

Для локального запуска или деплоя на сервер используется `docker-compose`.

### 1. Запуск Backend и Базы Данных
Убедитесь, что у вас установлен Docker и Docker Compose.

```bash
cd backend
docker-compose up -d --build
```
После запуска контейнеров выполните миграции и создайте администратора:
```bash
docker-compose exec web python manage.py migrate
docker-compose exec web python manage.py createsuperuser
```
*API будет доступно по адресу `http://localhost:8000/api/`*

### 2. Запуск Web-Панели (React)
Требуется Node.js v18+.
```bash
cd admin-web
npm install
npm start
```
*Панель администратора откроется на `http://localhost:3000`*

### 3. Запуск Мобильного приложения (Flutter)
Требуется Flutter SDK.
```bash
cd media_app
flutter pub get
flutter run
```

## Конфигурация

Перед публикацией на production-сервере необходимо настроить переменные в файле `backend/core/settings.py` и `backend/events/views.py`:

1.  **Секретный ключ:** Замените `SECRET_KEY` в Django на надежный случайный хэш.
2.  **Домены:** Добавьте ваши домены в список `ALLOWED_HOSTS`.
3.  **Telegram Bot:** Впишите токен от BotFather в переменную `TELEGRAM_BOT_TOKEN` в файле `views.py` для активации уведомлений.
4.  **База данных:** Обязательно смените дефолтные пароли для PostgreSQL в `docker-compose.yml` и `settings.py`.
5.  **API URL:** Укажите боевой URL бэкенда в файлах `admin-web/src/services/api.ts` и `media_app/lib/services/api_service.dart`.
